# 项目推进时间线（Timeline）

说明：以周为粒度，按阶段推进。可按实际日期替换“第X周”。

相关文档
- 需求分析: `doc/requirements-analysis.md`
- 数据库说明: `doc/database.md`
- 微服务拓扑: `doc/microservices-topology.md`
- 接口定义: `doc/openapi.yaml`
- 部署手册: `deployment-guide.md`

## 第1周：需求与范围
- 需求梳理与角色确认（用户/管理员/内容/统计）
- 核心业务流程与验收标准
- 风险识别与里程碑定义
- 产出：需求分析文档、验收标准、里程碑

## 第2周：架构与数据建模
- 微服务拆分与接口边界
- 数据库表结构设计与初始化脚本
- 关键字段与状态枚举确认
- 产出：架构图、DB Schema、初始化脚本

## 第3-4周：后端基础能力
- 身份认证与用户管理
- 内容发布/审核/删除
- 搜索与基础统计接口
- 产出：核心 API、单测/自测记录

## 第5-6周：前端管理端与用户端
- 管理端：用户管理、内容管理、审核流
- 用户端：内容浏览、发布、互动
- 统计面板与图表联调
- 产出：可演示前端、接口联调记录

## 第7周：部署与验收
- Docker/compose 部署链路
- Nginx 反向代理与跨域
- 数据初始化与迁移说明
- 产出：部署文档、演示环境

## 第8周：收尾与交付
- 修复遗留问题
- 使用说明与答辩材料
- 产出：最终报告与演示视频
